# AI 面试模块工作台重设计方案

- 日期：2026-04-23
- 状态：已定稿
- 范围：AI 面试模块非模拟面试界面重设计
- 保留项：`InterviewSimulationPanel.vue` 深色沉浸式模拟面试主战场保持不改

## 1. 背景

当前 AI 面试模块已经具备较完整的功能链路，包括岗位准备、模拟面试、专项训练、面试复盘等阶段，但非模拟面试部分存在以下问题：

1. 信息架构偏“功能堆叠”，用户很难一眼判断当前阶段的主任务。
2. 标题层级、卡片密度、间距节奏和视觉主次不统一，导致扫描效率低。
3. 历史记录、快捷入口、结果洞察、主操作经常处于同一视觉层级，注意力竞争严重。
4. 非模拟面试界面与全站浅色工作台风格衔接不够稳定，和深色模拟面试区的边界也不够清晰。
5. 当前模块内局部样式语言过多，维护成本高，设计 token 文件也存在编码异常风险。

本次设计目标不是在现有界面上修补，而是重新建立一套更适合 AI 面试模块的工作台式布局、视觉系统和交互规则。

## 2. 设计目标

### 2.1 业务目标

1. 提升用户在“准备 -> 训练 -> 复盘”链路中的操作清晰度。
2. 让每个阶段都围绕一个明确主任务展开，减少界面干扰。
3. 提高 AI 面试模块和其他模块的整体风格一致性。
4. 为后续实现和扩展提供稳定的组件与 token 基础。

### 2.2 体验目标

1. 非模拟面试区统一为浅色专业工作台。
2. 模拟面试区保留深色沉浸式空间，形成明确的双空间分工。
3. 标题、模块、按钮、状态、反馈形成统一视觉语言。
4. 至少满足 WCAG AA 级别的基础可访问性要求。

## 3. 设计原则

1. 单屏单主任务：每个页面只保留一个最明确的主操作。
2. 主任务优先：主内容永远放在中央主列，上下文信息退居侧栏。
3. 流程连续：岗位准备、专项训练、面试复盘之间切换要保留上下文。
4. 系统一致：继承全站蓝灰浅色工作台风格，不另起一套割裂品牌语言。
5. 深浅分区：浅色负责准备和分析，深色负责模拟和沉浸。
6. 克制而高级：减少无意义装饰，重点用标题层级、留白和模块秩序建立高级感。

## 4. 适用范围与落点

### 4.1 重设计范围

以下界面需要重新设计布局、UI 和交互：

1. `src/components/ai/interview/AiInterviewerPanel.vue`
2. `src/components/ai/interview/JdAnalysisStage.vue`
3. `src/components/ai/interview/InterviewDrillPanel.vue`
4. `src/components/ai/interview/InterviewReviewPanel.vue`
5. `src/components/ai/interview/PrepConfigDialog.vue`
6. `src/components/ai/interview/ResumePreviewOverlay.vue`
7. `src/assets/interview-tokens.css`

### 4.2 保留范围

以下界面本轮不重做，仅要求后续检查衔接一致性：

1. `src/components/ai/interview/InterviewSimulationPanel.vue`

## 5. 总体架构

本次设计将 AI 面试模块拆成两个视觉空间：

| 空间 | 主题 | 职责 |
|---|---|---|
| 非模拟面试空间 | 浅色工作台 | 岗位准备、专项训练、面试复盘、配置、预览 |
| 模拟面试空间 | 深色沉浸式 | 实时模拟面试、语音交互、舞台感反馈 |

整体结构由“统一工作台外壳 + 阶段化主内容区 + 固定右侧上下文栏”构成。

## 6. 信息架构

### 6.1 页面层级

AI 面试模块非模拟区重组为以下 4 个层级：

| 层级 | 页面职责 | 核心内容 |
|---|---|---|
| L1 | AI 面试总工作台 | 阶段导航、当前状态、快捷恢复、全局摘要 |
| L2 | 岗位准备 | JD 输入、岗位解析、岗位洞察、开始配置 |
| L2 | 专项训练 | 训练题、即时反馈、薄弱项、训练模式切换 |
| L2 | 面试复盘 | 评分结论、能力诊断、岗位匹配、简历联动建议 |

### 6.2 顶层工作台结构

非模拟态 `AiInterviewerPanel.vue` 采用以下骨架：

1. 顶部工作台头部
2. 中央主任务区
3. 右侧上下文栏

#### 顶部工作台头部

- 左侧：动态主标题、副标题、当前岗位标签
- 中间：阶段导航 `岗位准备 / 模拟面试 / 专项训练 / 面试复盘`
- 右侧：当前阶段唯一主操作按钮

#### 中央主任务区

- 只展示当前阶段的核心任务，不同时堆叠多个平级模块

#### 右侧上下文栏

- 当前岗位摘要
- 面试模式与时长
- 最近一次记录
- 快捷操作
- 小型状态卡

## 7. 视觉设计系统

### 7.1 视觉基调

非模拟面试区延续全站主风格：

1. 蓝灰主品牌色
2. 浅色工作台底色
3. 轻雾面卡片与柔和圆角
4. 信息密度适中但不拥挤

避免使用紫色默认审美、过重发光、过强舞台感或与全站明显割裂的未来风。

### 7.2 色彩系统

#### 主色

- 主按钮、链接、焦点态使用品牌蓝
- 阶段激活态、关键标签使用低饱和蓝灰过渡

#### 中性色层级

1. App 背景层：最浅灰蓝底
2. 工作台层：浅白雾面底
3. 卡片层：高纯度白底
4. 强调层：浅品牌色或轻抬升背景

#### 语义色

- 成功：偏青绿
- 警示：低饱和琥珀
- 风险：偏砖红
- 信息：品牌蓝浅变体

### 7.3 标题层级

| 层级 | 用途 | 建议规格 |
|---|---|---|
| H1 | 页面主标题 | 28-32px，高字重，配任务型副标题 |
| H2 | 大模块标题 | 20-24px |
| H3 | 卡片组标题 | 16-18px |
| Label | 元信息标签 | 12-13px，高字重 |

标题文案统一改为任务型表达，直接说明这块内容对用户的价值。

### 7.4 间距系统

采用 8 点间距体系，统一节奏：

- 小元素：8 / 12
- 卡片内部：16 / 20
- 卡片之间：24
- 大区块之间：32 / 40
- 页面头部与主体：32

### 7.5 卡片系统

统一收敛为 3 类卡片：

1. 主任务卡：用于输入、训练、结果主面板
2. 信息摘要卡：用于侧栏、状态摘要、记录概览
3. 强调卡：用于推荐下一步、风险提醒、关键结论

### 7.6 按钮系统

统一为四级：

1. Primary：唯一主动作
2. Secondary：辅助动作
3. Tertiary：文本按钮
4. Danger：风险动作

原则：同一视图只允许一个最高优先级主按钮。

## 8. 关键页面布局

### 8.1 AI 面试总工作台

`AiInterviewerPanel.vue` 非模拟态改造目标：

1. 从“多面板容器”升级为“任务中枢”
2. 通过顶部动态标题和阶段导航强化流程位置感
3. 通过固定右侧上下文栏承载参考信息，降低主视区负担

### 8.2 岗位准备页

`JdAnalysisStage.vue` 改成“两段式页面”：

#### 第一段：准备入口

- 左主区：标题、JD 输入、开始按钮、跳过按钮
- 右辅区：最近记录、最近一次 JD 摘要、准备状态提示

#### 第二段：解析结果工作台

解析完成后，主区切换为结果态：

1. 这个岗位最看重什么
2. 你的简历还缺什么
3. 优先讲哪几个项目故事
4. 面试官最可能深挖什么

底部动作条统一提供：

- 配置模拟面试
- 进入专项训练
- 稍后继续

历史记录从主视区移出，改为侧栏或抽屉。

### 8.3 专项训练页

`InterviewDrillPanel.vue` 改造为训练工作区：

- 左侧：训练标题、当前题目、回答区、即时点评
- 右侧：训练方向、进度、薄弱项、推荐下一题
- 顶部：训练模式切换 `项目深挖 / 原理问答 / 行为面试 / 系统设计`

### 8.4 面试复盘页

`InterviewReviewPanel.vue` 改造成诊断报告页：

#### 第一屏：诊断总览

- 总评分
- 是否通过
- 一句话总结
- 三个关键判断

#### 第二屏：三栏诊断

1. 回答表现
2. 岗位匹配
3. 简历联动建议

聊天记录不再默认大面积展开，只作为折叠区提供。

### 8.5 配置弹窗

`PrepConfigDialog.vue` 从密集参数弹窗改为轻量步骤式配置层：

1. 面试模式
2. 模型选择
3. 时长
4. 重点考察方向
5. 自定义备注

底部固定动作区：

- 开始这轮面试
- 返回调整
- 查看简历摘要

### 8.6 简历预览层

`ResumePreviewOverlay.vue` 从覆盖式弹层改为右侧抽屉更合适：

- 顶部：简历概览
- 中部：关键项目与技能
- 底部：AI 建议重点追问点

## 9. 交互规则与状态设计

### 9.1 交互总原则

1. 单屏单主任务
2. 上下文不打断流程
3. 状态先于内容
4. 动作有后果预期

### 9.2 阶段导航规则

每个阶段统一 4 种状态：

1. 未开始
2. 进行中
3. 已完成
4. 需关注

切换要求：

1. 缺少上下文时给轻提示，不直接跳空页
2. 模拟面试结束后默认进入复盘
3. 从复盘返回训练时保留当前岗位上下文

### 9.3 页面状态

#### 空状态

必须说明：

1. 为什么为空
2. 可以做什么

#### 加载状态

需要语义化反馈，例如：

- 正在提炼岗位核心要求
- 正在生成高风险追问
- 正在汇总本轮面试诊断

#### 错误状态

至少提供：

1. 哪一步失败
2. 可能原因
3. 下一步可选动作

#### 完成状态

明确给出推荐下一步，而不是只显示“已完成”。

### 9.4 容器使用边界

#### 抽屉

适用于参考信息：

- 简历预览
- 历史记录详情
- 岗位摘要扩展内容

#### 弹窗

适用于必须确认的任务：

- 面试前配置
- 删除确认
- 覆盖确认

#### 页内展开

适用于补充细节：

- 原始聊天记录
- 详细分析说明
- 次级建议项

### 9.5 反馈闭环

专项训练与复盘页面都必须把结果映射成下一步行动：

1. 发现了什么
2. 为什么重要
3. 用户下一步该练什么或改什么

## 10. 可访问性要求

至少满足以下要求：

1. 色彩对比达到 WCAG AA
2. 所有关键操作具有清晰焦点态
3. 状态不只依赖颜色区分
4. 抽屉和弹窗支持键盘关闭与焦点回收
5. 窄屏下内容可重排，不出现不可达区域

## 11. 技术设计建议

### 11.1 Token 重建

建议重构 `src/assets/interview-tokens.css`：

1. 修复编码异常
2. 拆分浅色工作台 token 与深色模拟面试 token
3. 用语义 token 替代局部魔法值
4. 让卡片、按钮、标签、输入区共享统一基础变量

### 11.2 组件职责收敛

建议后续实现时控制职责边界：

1. `AiInterviewerPanel.vue` 负责工作台壳层与阶段编排
2. `JdAnalysisStage.vue` 负责岗位准备页
3. `InterviewDrillPanel.vue` 负责训练工作区
4. `InterviewReviewPanel.vue` 负责复盘报告页
5. `PrepConfigDialog.vue` 负责出发前配置
6. `ResumePreviewOverlay.vue` 负责侧边预览层

## 12. 验收标准

### 12.1 视觉验收

1. 非模拟面试区统一为浅色工作台
2. 模拟面试区继续保持深色沉浸式
3. 标题、卡片、按钮、间距形成统一系统
4. 与其他模块的品牌色和整体气质一致

### 12.2 布局验收

1. 当前页面主任务在第一视区可明确识别
2. 历史记录和辅助信息不再占据主视区
3. 右侧上下文栏承载稳定，不和主内容抢权重
4. 岗位准备、训练、复盘三类页面布局差异清晰但语言统一

### 12.3 交互验收

1. 每页只有一个最明确主按钮
2. 状态反馈完整，空态/加载/错误/完成均有语义化设计
3. 抽屉、弹窗、页内展开使用边界清晰
4. 阶段切换保留上下文，不造成割裂感

## 13. 后续实施建议

建议按以下顺序实施：

1. 先重建 token 和工作台壳层
2. 再重做岗位准备页
3. 再重做专项训练页与复盘页
4. 最后统一配置层、预览层和响应式细节

---

**结论**：本次方案的本质不是局部美化，而是把 AI 面试非模拟区重构为一套浅色专业工作台体系，同时保留深色模拟面试主战场，形成更适合求职面试场景的“双空间系统”。
